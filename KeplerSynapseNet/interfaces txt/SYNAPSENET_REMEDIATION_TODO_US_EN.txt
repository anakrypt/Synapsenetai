    Current repo state:
    - Confirmed in `KeplerSynapseNet/include/core/wallet.h` and
      `KeplerSynapseNet/src/core/wallet.cpp`.
    - Wallet API and storage still use `double` for `balance`,
      `pendingBalance`, and `stakedBalance`.
    - PoE already uses atom-based stake semantics in the main runtime path
      (`validatorMinStakeAtoms`).
    Required work:
    - Introduce a canonical atomic unit type, for example `uint64_t AmountAtoms`.
    - Define one public constant for conversion, for example
      `1 NGT = 100000000 atoms`.
    - Replace all wallet/core balance fields, getters, setters, serializers,
      RPC payloads, and TUI formatting paths that currently use `double`.
    - Keep decimal formatting only at UI and RPC presentation boundaries.
    - Add migration logic for any persisted state that serialized balances as
      floating-point values.
    Acceptance criteria:
    - No financial state in wallet/core code uses `double`.
    - Internal transfer, staking, pending-balance, and reward logic use atoms.
    - RPC and TUI expose both human-readable NGT and raw atoms where needed.
    - Tests cover large values, zero, rounding boundaries, and serialization.
